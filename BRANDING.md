```

    ║  ▄▄▄▄▄▄▄▄▄▄▄   ║
```

## Design System

### Buttons

#### Primary Button
```css
background: linear-gradient(135deg, #7C3AED 0%, #5B21B6 100%);
color: #FFFFFF;
border: none;
border-radius: 12px;
padding: 12px 24px;
font-weight: 600;
transition: all 0.2s ease;
box-shadow: 0 4px 14px rgba(124, 58, 237, 0.4);
```

**States:**
- Hover: `transform: translateY(-2px); box-shadow: 0 6px 20px rgba(124, 58, 237, 0.5);`
- Active: `transform: translateY(0);`
- Focus: `outline: 2px solid #00D4FF; outline-offset: 2px;`
- Disabled: `opacity: 0.5; cursor: not-allowed;`

**Tailwind:**
```html
<button class="bg-gradient-to-r from-purple-600 to-purple-700 text-white
               font-semibold py-3 px-6 rounded-xl
               shadow-lg shadow-purple-500/40
               hover:-translate-y-0.5 hover:shadow-xl hover:shadow-purple-500/50
               active:translate-y-0
               focus:outline-none focus:ring-2 focus:ring-cyan-400 focus:ring-offset-2
               disabled:opacity-50 disabled:cursor-not-allowed
               transition-all duration-200">
  Start Learning
</button>
```

## Animation Guidelines

### Micro-interactions
- **Button hover:** `0.2s` ease-out transform
- **Card hover:** `0.3s` ease-out with shadow
- **Load:** Skeleton with shimmer effect
- **Success:** Checkmark animation with bounce
- **Error:** Shake animation

### Page Transitions
- **Fade in:** `opacity: 0 → 1` over `0.3s`
- **Slide up:** `translateY(20px) → 0` over `0.4s`
- **Stagger:** Children animate with `50ms` delay

### XP Gain Animation

```css
@keyframes xpFloat {
  0% { opacity: 1; transform: translateY(0) scale(1); }
  100% { opacity: 0; transform: translateY(-50px) scale(1.5); }
}

}
```

## Do's and Don'ts

### Do
✓ Use gradients for CTAs and highlights
✓ Add subtle glow effects to important elements
✓ Use dark backgrounds with high contrast
✓ Keep animations smooth (300ms or less)
✓ Use gaming terminology correctly
✓ Show progress visually (XP bars, ranks)

### Don't
✗ Use childish illustrations or cartoons
✗ Overdo gradients (max 2 colors)
✗ Make animations too slow or distracting
✗ Use low contrast (accessibility matters)
✗ Mix too many colors (max 3 per element)
✗ Ignore mobile responsiveness

## Accessibility

### Color Contrast
- All text must meet WCAG AA (4.5:1)
- Primary text: #F8F8FC on #0A0A0F = 16.6:1 ✓
- Secondary text: #A1A1AA on #0A0A0F = 6.8:1 ✓

### Focus States
- 2px outline on all interactive elements
- Outline color: #00D4FF (Cyan)
- Always visible on keyboard navigation

### Screen Readers
- Use semantic HTML
- ARIA labels for icons
- Alt text for images
- Announce dynamic content (XP gains, level ups)
